## 🎯 Key Principles

1. **Simplicity First** - Regex parsing over complex AST libraries
2. **Readable Code** - Every line should be understandable
3. **Incremental Development** - One feature at a time
4. **Real-World Testing** - Validate with actual examples

## 📖 For Newcomers

1. Start with [Phase 1](phases/phase1-proof-of-concept.md) to understand the core concept
2. Read [Phase Overview](phases/README.md) to see the full journey
3. Check [Architecture Evolution](phases/architecture-evolution.md) to understand refactoring decisions

## 🔧 For Contributors

1. Understand the phase-based development approach
2. Follow the established patterns in `src/` directory
3. Add tests that match the architectural layers
